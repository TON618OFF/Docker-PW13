## 1. НАЗНАЧЕНИЕ ПРОГРАММЫ

Веб-приложение «ImperialTunes» предназначено для удобного и быстрого доступа к музыкальному контенту через современный веб-интерфейс. Основные функции приложения включают просмотр каталога музыкальных треков, альбомов и артистов, воспроизведение музыки, создание и управление плейлистами, загрузку собственного контента для артистов, а также управление пользователями и контентом для администраторов и модераторов.

Приложение ориентировано на любителей музыки, которые хотят прослушивать музыкальные треки, создавать плейлисты и управлять своей музыкальной библиотекой, на артистов, которые хотят загружать и распространять свой контент, а также на администраторов, отвечающих за управление контентом и пользователями. Оно предоставляет интуитивно понятный интерфейс и высокую производительность, обеспечивая удобство использования для широкого круга пользователей.

Приложение поддерживает различные роли пользователей:
- **Слушатель** — базовый пользователь, может прослушивать музыку, создавать плейлисты, добавлять треки в избранное
- **Артист** — может загружать треки и альбомы, управлять своим контентом, просматривать статистику
- **Дистрибьютор** — может обрабатывать заявки на статус артиста, управлять контентом
- **Модератор** — может модерировать контент, обрабатывать заявки
- **Администратор** — имеет полный доступ к управлению системой, пользователями и контентом

## 2. УСЛОВИЯ ВЫПОЛНЕНИЯ ПРОГРАММЫ

### 2.1. Системные требования

Веб-приложение «ImperialTunes» работает в современных веб-браузерах и не требует установки на устройство. Приложение доступно через веб-браузер по URL-адресу.

**Минимальные системные требования:**

**Оборудование:**
- Процессор: с частотой не менее 1 ГГц
- Оперативная память: не менее 2 ГБ
- Свободное место на диске: не требуется (приложение работает в браузере)
- Звуковая карта: для воспроизведения аудио
- Интернет-соединение: стабильное подключение к интернету (скорость не менее 1 Мбит/с)

**Программное обеспечение:**
- Операционная система: Windows 10/11, macOS 10.14+, Linux (любая современная дистрибуция), Android 8.0+, iOS 12.0+
- Веб-браузер: Google Chrome 90+, Mozilla Firefox 88+, Microsoft Edge 90+, Safari 14+, Opera 76+

**Рекомендуемые системные требования:**

**Оборудование:**
- Процессор: с частотой не менее 2 ГГц (многоядерный)
- Оперативная память: не менее 4 ГБ
- Интернет-соединение: стабильное подключение к интернету (скорость не менее 5 Мбит/с для комфортного прослушивания музыки)

**Программное обеспечение:**
- Веб-браузер: последняя версия Google Chrome, Mozilla Firefox или Microsoft Edge

## 3. ВЫПОЛНЕНИЕ ПРИЛОЖЕНИЯ

### 3.1. Действия для доступа к приложению

Веб-приложение «ImperialTunes» не требует установки на устройство. Для доступа к приложению необходимо:

1. Открыть веб-браузер на устройстве (Google Chrome, Mozilla Firefox, Microsoft Edge, Safari и т.д.)

2. Ввести URL-адрес приложения в адресную строку браузера или перейти по предоставленной ссылке

3. Дождаться загрузки приложения (первые несколько секунд может потребоваться для загрузки ресурсов)

4. Если пользователь не авторизован, будет автоматически отображена страница авторизации/регистрации

5. После авторизации пользователь попадает на главную страницу приложения

**Примечание:** Для первого запуска приложения требуется подключение к интернету, чтобы загрузить данные о треках, альбомах, артистах и пользовательских настройках из базы данных Supabase. После первоначальной загрузки некоторые данные могут кэшироваться в браузере, но для полноценной работы приложения (прослушивание музыки, загрузка контента, обновление данных) необходимо постоянное подключение к интернету.

### 3.2. Действия для запуска приложения

После доступа к приложению через веб-браузер приложение автоматически загружается и отображает интерфейс. Если пользователь уже авторизован в системе, он сразу попадает на главную страницу. Если пользователь не авторизован, он перенаправляется на страницу авторизации.

### 3.3. Выполнение приложения с описанием функций

#### 3.3.1. Главная страница

При запуске приложения авторизованный пользователь видит главную страницу, которая отображает обзор музыкального каталога, популярные треки, недавно добавленные альбомы и артистов. На главной странице также отображается навигационное меню для перехода между основными разделами приложения.

**Основные элементы главной страницы:**

- **Заголовок приложения** — логотип и название «ImperialTunes» в верхней части страницы
- **Навигационное меню** — ссылки на основные разделы: «Главная», «Библиотека», «Плейлисты», «Аналитика», «Профиль», «Настройки»
- **Статистика** — общее количество треков, плейлистов и прослушиваний
- **Популярные треки** — список наиболее популярных треков с возможностью быстрого воспроизведения
- **Недавние альбомы** — список недавно добавленных альбомов
- **Музыкальный плеер** — нижняя панель с элементами управления воспроизведением (отображается при воспроизведении трека)

**Функции главной страницы:**

- **Быстрое воспроизведение** — нажатие на кнопку «Воспроизвести» на любом треке запускает его воспроизведение в музыкальном плеере
- **Переход к деталям** — нажатие на карточку трека, альбома или артиста открывает страницу с подробной информацией
- **Навигация** — клик по элементам навигационного меню перенаправляет на соответствующий раздел приложения

#### 3.3.2. Авторизация и регистрация

Для использования приложения пользователь должен быть авторизован в системе. Если пользователь не авторизован, он автоматически перенаправляется на страницу авторизации.

**Регистрация нового пользователя:**

1. На странице авторизации переключиться на режим регистрации (кнопка «Зарегистрироваться»)

2. Заполнить форму регистрации:
   - **Email** — ввести действующий адрес электронной почты
   - **Пароль** — ввести пароль (минимум 6 символов)
   - **Имя пользователя** — ввести уникальное имя пользователя (от 3 до 50 символов)
   - **Имя** (опционально) — ввести имя
   - **Фамилия** (опционально) — ввести фамилию

3. Нажать кнопку «Зарегистрироваться»

4. После успешной регистрации пользователь автоматически авторизуется и перенаправляется на главную страницу

5. Новому пользователю автоматически назначается роль «слушатель» по умолчанию

**Авторизация существующего пользователя:**

1. На странице авторизации ввести email и пароль

2. Нажать кнопку «Войти»

3. После успешной авторизации пользователь перенаправляется на главную страницу

4. Поле `last_login` в базе данных обновляется с текущей датой и временем

**Восстановление пароля:**

1. На странице авторизации нажать на ссылку «Забыли пароль?»

2. Ввести email зарегистрированного пользователя

3. Нажать кнопку «Отправить»

4. На указанный email будет отправлено письмо с инструкциями по восстановлению пароля

5. Перейти по ссылке из письма и следовать инструкциям для сброса пароля

#### 3.3.3. Раздел «Библиотека»

Раздел «Библиотека» предоставляет доступ к полному каталогу музыкальных треков, альбомов и артистов. Пользователь может просматривать, искать, фильтровать и сортировать музыкальный контент.

**Основные функции раздела «Библиотека»:**

**Просмотр каталога треков:**

- Список всех доступных треков отображается в виде карточек
- Каждая карточка содержит: название трека, имя артиста, название альбома, длительность, количество прослушиваний, кнопки для взаимодействия
- Треки отображаются только если они публичные (`is_public = true`)

**Поиск музыкального контента:**

- В верхней части страницы расположена строка поиска
- Пользователь может ввести запрос для поиска треков, альбомов или артистов
- Поиск работает динамически, фильтруя результаты в реальном времени
- Поиск выполняется по названиям треков, альбомов и артистов

**Фильтрация и сортировка:**

- Кнопка «Фильтр» открывает окно с параметрами фильтрации и сортировки
- Доступные параметры сортировки:
  - По дате добавления (по возрастанию/убыванию)
  - По названию (по алфавиту)
  - По количеству прослушиваний (по популярности)
- Доступные параметры фильтрации:
  - По жанрам (выбор одного или нескольких жанров)
  - По артистам
  - По альбомам

**Добавление в избранное:**

- Кнопка с иконкой «сердце» на карточке трека позволяет добавить трек в избранное
- Если трек уже в избранном, иконка отображается заполненной
- Добавление и удаление из избранного выполняется через функцию `toggle_favorite_track`

**Воспроизведение треков:**

- Кнопка «Воспроизвести» на карточке трека запускает его воспроизведение в музыкальном плеере
- При воспроизведении трека автоматически записывается история прослушиваний в базу данных
- Счётчик прослушиваний (`track_play_count`) увеличивается автоматически

**Переход к деталям:**

- Нажатие на карточку трека открывает страницу с подробной информацией о треке
- На странице деталей отображается: полная информация о треке, список жанров, информация об альбоме и артисте, возможность добавить трек в плейлист

**Управление контентом (для артистов):**

- Если пользователь имеет роль «артист», в разделе «Библиотека» доступна вкладка «Мои треки»
- На вкладке «Мои треки» артист может:
  - Просматривать свои загруженные треки
  - Загружать новые треки
  - Редактировать существующие треки
  - Удалять треки
  - Управлять альбомами

#### 3.3.4. Раздел «Плейлисты»

Раздел «Плейлисты» позволяет пользователям создавать, редактировать и управлять плейлистами. Пользователи могут создавать публичные и приватные плейлисты, добавлять в них треки и делиться плейлистами с другими пользователями.

**Основные функции раздела «Плейлисты»:**

**Просмотр плейлистов:**

- Список всех доступных плейлистов отображается в виде карточек
- Каждая карточка содержит: название плейлиста, обложку, имя создателя, количество треков, длительность плейлиста, количество подписчиков
- Отображаются как собственные плейлисты пользователя, так и публичные плейлисты других пользователей

**Создание плейлиста:**

1. Нажать кнопку «Создать плейлист» в верхней части страницы
2. В открывшемся диалоговом окне заполнить форму:
   - **Название плейлиста** — ввести название (обязательное поле)
   - **Описание** (опционально) — ввести описание плейлиста
   - **Видимость** — выбрать «Публичный» или «Приватный»
   - **Обложка** (опционально) — загрузить изображение обложки
3. Нажать кнопку «Создать»
4. После создания плейлист отображается в списке плейлистов пользователя

**Редактирование плейлиста:**

1. Открыть страницу плейлиста, нажав на его карточку
2. На странице плейлиста нажать кнопку «Редактировать»
3. Изменить название, описание или видимость плейлиста
4. Сохранить изменения

**Добавление треков в плейлист:**

1. На странице плейлиста нажать кнопку «Добавить треки»
2. В открывшемся диалоговом окне выбрать треки из каталога
3. Нажать кнопку «Добавить»
4. Треки добавляются в плейлист в выбранном порядке

**Управление треками в плейлисте:**

- На странице плейлиста отображается список всех треков
- Пользователь может:
  - Удалить трек из плейлиста (кнопка «Удалить»)
  - Изменить порядок треков (перетаскивание)
  - Воспроизвести плейлист (кнопка «Воспроизвести весь плейлист»)

**Добавление плейлиста в избранное:**

- Кнопка с иконкой «сердце» на карточке плейлиста позволяет добавить плейлист в избранное
- Добавление и удаление из избранного выполняется через функцию `toggle_favorite_playlist`

#### 3.3.5. Музыкальный плеер

Музыкальный плеер отображается в нижней части экрана при воспроизведении трека. Плеер предоставляет полный контроль над воспроизведением музыки.

**Основные элементы музыкального плеера:**

- **Информация о треке** — название трека, имя артиста, обложка альбома
- **Кнопки управления** — воспроизведение/пауза, переключение на предыдущий/следующий трек
- **Прогресс-бар** — отображает текущую позицию воспроизведения и общую длительность трека
- **Регулировка громкости** — ползунок для изменения громкости звука
- **Дополнительные функции** — перемешивание, повтор, добавление в избранное

**Функции музыкального плеера:**

**Воспроизведение и пауза:**

- Кнопка «Воспроизвести» запускает воспроизведение текущего трека
- Кнопка «Пауза» приостанавливает воспроизведение
- Воспроизведение можно управлять с помощью клавиши пробела на клавиатуре

**Переключение треков:**

- Кнопка «Предыдущий трек» переключает на предыдущий трек в плейлисте
- Кнопка «Следующий трек» переключает на следующий трек в плейлисте
- Переключение можно управлять с помощью стрелок на клавиатуре

**Перемотка:**

- Перетаскивание ползунка прогресс-бара позволяет перемотать трек на нужную позицию
- Отображается текущее время и общая длительность трека

**Регулировка громкости:**

- Ползунок громкости позволяет изменить уровень звука от 0% до 100%
- Кнопка «Отключить звук» позволяет быстро отключить/включить звук
- Громкость можно управлять с помощью клавиши M на клавиатуре

**Перемешивание и повтор:**

- Кнопка «Перемешать» включает/выключает режим случайного воспроизведения треков
- Кнопка «Повтор» включает/выключает режим повторения (нет повторения, повтор одного трека, повтор всего плейлиста)

**Добавление в избранное:**

- Кнопка с иконкой «сердце» позволяет добавить текущий трек в избранное или удалить его из избранного

**Автоматическая запись истории:**

- При воспроизведении трека автоматически создаётся запись в таблице `listening_history`
- Записывается: ID пользователя, ID трека, дата и время прослушивания, длительность прослушивания
- Счётчик прослушиваний (`track_play_count`) увеличивается автоматически через триггер

#### 3.3.6. Раздел «Профиль»

Раздел «Профиль» позволяет пользователям просматривать и редактировать свои данные, просматривать статистику и управлять своими плейлистами и избранным.

**Основные функции раздела «Профиль»:**

**Просмотр профиля:**

- Отображается информация о пользователе: имя пользователя, имя, фамилия, email, роль, биография, аватар, дата регистрации, дата последнего входа
- Отображается статистика: количество загруженных треков, количество созданных плейлистов, количество элементов в избранном

**Редактирование профиля:**

1. На странице профиля нажать кнопку «Редактировать профиль»
2. В открывшемся диалоговом окне изменить данные:
   - Имя пользователя
   - Имя
   - Фамилию
   - Биографию
   - Аватар (загрузка изображения)
3. Нажать кнопку «Сохранить»
4. Изменения сохраняются в базе данных

**Просмотр своих треков:**

- Раздел «Мои треки» отображает список всех треков, загруженных пользователем (для артистов)
- На странице можно:
  - Просматривать загруженные треки
  - Загружать новые треки
  - Редактировать существующие треки
  - Удалять треки

**Просмотр своих плейлистов:**

- Раздел «Мои плейлисты» отображает список всех плейлистов, созданных пользователем
- На странице можно:
  - Просматривать плейлисты
  - Создавать новые плейлисты
  - Редактировать плейлисты
  - Удалять плейлисты

**Просмотр избранного:**

- Раздел «Моё избранное» отображает список всех треков, альбомов и плейлистов, добавленных в избранное
- На странице можно:
  - Просматривать избранные элементы
  - Удалять элементы из избранного
  - Воспроизводить избранные треки

**История прослушиваний:**

- На странице профиля отображается история прослушиваний пользователя
- История показывает: название трека, имя артиста, дату и время прослушивания, длительность прослушивания
- Пользователь может воспроизвести трек из истории, нажав на него

**Подача заявки на статус артиста:**

- Если пользователь имеет роль «слушатель», на странице профиля доступна форма «Стать артистом»
- Для подачи заявки необходимо:
  1. Заполнить форму: название артиста/группы, описание, контактная информация
  2. Отправить заявку
  3. Дождаться рассмотрения заявки дистрибьютором или модератором
  4. После одобрения заявки пользователю назначается роль «артист»

#### 3.3.7. Раздел «Аналитика»

Раздел «Аналитика» предоставляет пользователям возможность просматривать статистику прослушиваний, популярности треков и активности пользователей. Для артистов доступна статистика по их загруженным трекам.

**Основные функции раздела «Аналитика»:**

**Общая статистика:**

- Отображается общая статистика прослушиваний: общее количество прослушиваний, общая длительность прослушиваний, средняя длительность трека, количество треков

**Графики прослушиваний:**

- График прослушиваний по времени показывает количество прослушиваний за выбранный период (день, неделя, месяц)
- График отображается в виде столбчатой диаграммы или линейного графика
- Графики создаются с использованием библиотеки Recharts

**Статистика по жанрам:**

- Круговая диаграмма показывает распределение прослушиваний по жанрам
- Отображается процент прослушиваний для каждого жанра

**Популярные треки:**

- Список наиболее популярных треков с количеством прослушиваний
- Треки отсортированы по количеству прослушиваний

**Фильтры:**

- Фильтр по датам позволяет выбрать период для отображения статистики
- Фильтр по жанрам позволяет отфильтровать статистику по выбранным жанрам
- Фильтр по артистам позволяет отфильтровать статистику по выбранным артистам

**Экспорт данных:**

- Кнопка «Экспорт» позволяет экспортировать данные аналитики в формате CSV или JSON
- Экспортированные данные можно использовать для дальнейшего анализа

**Статистика для артистов:**

- Если пользователь имеет роль «артист», в разделе «Аналитика» отображается статистика по его загруженным трекам
- Статистика включает: количество прослушиваний, популярность треков, статистику по жанрам, графики прослушиваний

#### 3.3.8. Загрузка контента (для артистов)

Если пользователь имеет роль «артист», он может загружать свои треки и создавать альбомы. Загрузка контента доступна через раздел «Библиотека» на вкладке «Мои треки» или через раздел «Профиль».

**Загрузка трека:**

1. На странице «Мои треки» нажать кнопку «Загрузить трек»
2. В открывшемся диалоговом окне заполнить форму:
   - **Название трека** — ввести название (обязательное поле)
   - **Альбом** — выбрать альбом из списка или оставить пустым
   - **Жанры** — выбрать один или несколько жанров
   - **Аудиофайл** — загрузить аудиофайл в формате MP3, WAV, FLAC, OGG или M4A
3. Нажать кнопку «Загрузить»
4. Аудиофайл загружается в Supabase Storage (bucket 'tracks')
5. Метаданные трека сохраняются в базу данных
6. Трек отображается в списке «Мои треки»

**Создание альбома:**

1. На странице «Мои треки» нажать кнопку «Создать альбом»
2. В открывшемся диалоговом окне заполнить форму:
   - **Название альбома** — ввести название (обязательное поле)
   - **Дата релиза** — выбрать дату релиза альбома
   - **Описание** (опционально) — ввести описание альбома
   - **Обложка** — загрузить изображение обложки альбома
3. Нажать кнопку «Создать»
4. Обложка альбома загружается в Supabase Storage (bucket 'albums')
5. Метаданные альбома сохраняются в базу данных
6. Альбом отображается в списке альбомов артиста

**Редактирование трека:**

1. На странице «Мои треки» выбрать трек для редактирования
2. Нажать кнопку «Редактировать»
3. В открывшемся диалоговом окне изменить данные:
   - Название трека
   - Альбом
   - Жанры
   - Публичность трека (публичный/приватный)
4. Нажать кнопку «Сохранить»
5. Изменения сохраняются в базе данных
6. Аудит-лог фиксирует изменения через триггер `audit_changes`

**Удаление трека:**

1. На странице «Мои треки» выбрать трек для удаления
2. Нажать кнопку «Удалить»
3. Подтвердить удаление
4. Трек удаляется из базы данных
5. Аудиофайл удаляется из Supabase Storage
6. Аудит-лог фиксирует удаление

#### 3.3.9. Административная панель

Административная панель доступна только для пользователей с ролью «администратор», «дистрибьютор» или «модератор». Панель предоставляет доступ к управлению пользователями, контентом, жанрами и заявками на статус артиста.

**Доступ к административной панели:**

1. В навигационном меню нажать кнопку «Административная панель» (доступна только для администраторов)
2. Если пользователь не имеет прав доступа, он перенаправляется на главную страницу
3. Если пользователь имеет права доступа, он попадает на главную страницу административной панели

**Управление пользователями:**

1. На главной странице административной панели нажать кнопку «Управление пользователями»
2. Отображается список всех пользователей системы
3. Для каждого пользователя доступны функции:
   - Просмотр профиля пользователя
   - Изменение роли пользователя
   - Блокировка/разблокировка пользователя (изменение статуса `is_active`)
   - Удаление пользователя

**Управление треками:**

1. На главной странице административной панели нажать кнопку «Управление треками»
2. Отображается список всех треков в системе
3. Для каждого трека доступны функции:
   - Просмотр деталей трека
   - Редактирование трека
   - Удаление трека
   - Изменение публичности трека

**Управление альбомами:**

1. На главной странице административной панели нажать кнопку «Управление альбомами»
2. Отображается список всех альбомов в системе
3. Для каждого альбома доступны функции:
   - Просмотр деталей альбома
   - Редактирование альбома
   - Удаление альбома (с каскадным удалением треков)

**Управление артистами:**

1. На главной странице административной панели нажать кнопку «Управление артистами»
2. Отображается список всех артистов в системе
3. Для каждого артиста доступны функции:
   - Просмотр профиля артиста
   - Редактирование информации об артисте
   - Удаление артиста

**Управление жанрами:**

1. На главной странице административной панели нажать кнопку «Управление жанрами»
2. Отображается список всех жанров в системе
3. Доступны функции:
   - Добавление нового жанра
   - Редактирование жанра (название, описание)
   - Удаление жанра (при условии, что жанр не используется в треках)

**Обработка заявок на статус артиста:**

1. В навигационном меню нажать кнопку «Заявки на артиста» (доступна для дистрибьюторов и модераторов)
2. Отображается список всех заявок на статус артиста
3. Для каждой заявки доступны функции:
   - Просмотр деталей заявки
   - Одобрение заявки (через функцию `approve_artist_application`)
   - Отклонение заявки (через функцию `reject_artist_application`)
4. При одобрении заявки:
   - Пользователю назначается роль «артист»
   - Создаётся запись артиста в таблице `artists`
   - Статус заявки изменяется на «одобрена»
5. При отклонении заявки:
   - Статус заявки изменяется на «отклонена»
   - Пользователю отправляется уведомление

**Просмотр аудит-логов:**

1. На главной странице административной панели нажать кнопку «Аудит-логи»
2. Отображается список всех записей аудит-лога
3. Доступны функции:
   - Просмотр деталей записи (старые и новые значения)
   - Фильтрация по типу операции (INSERT, UPDATE, DELETE)
   - Фильтрация по таблице
   - Фильтрация по пользователю
   - Фильтрация по дате

#### 3.3.10. Раздел «Настройки»

Раздел «Настройки» позволяет пользователям настраивать параметры приложения, изменять пароль, управлять избранным и настраивать тему оформления.

**Основные функции раздела «Настройки»:**

**Настройки профиля:**

- Редактирование данных профиля: имя пользователя, имя, фамилия, биография, аватар
- Изменение пароля:
  1. Ввести текущий пароль
  2. Ввести новый пароль

**Настройки приложения:**

- **Язык интерфейса** — выбор языка интерфейса (русский, английский)

- Просмотр избранных треков, альбомов и плейлистов

- Экспорт списка избранного в CSV или JSON

**Выход из аккаунта:**

- Кнопка «Выйти» позволяет выйти из аккаунта
- После выхода пользователь перенаправляется на страницу авторизации
- Сессия пользователя завершается

### 3.4. Горячие клавиши

Приложение поддерживает горячие клавиши для быстрого управления воспроизведением музыки и навигацией:

- **Пробел** — воспроизведение/пауза текущего трека
- **Стрелка влево** — переключение на предыдущий трек
- **Стрелка вправо** — переключение на следующий трек
- **M** — отключение/включение звука
- **Стрелка вверх** — увеличение громкости
- **Стрелка вниз** — уменьшение громкости

### 3.5. Мобильная версия

Приложение адаптировано для работы на мобильных устройствах (смартфоны и планшеты). На мобильных устройствах:

- Навигационное меню отображается в виде выдвижного меню (гамбургер-меню)
- Музыкальный плеер адаптирован под размер экрана
- Карточки треков, альбомов и плейлистов оптимизированы для сенсорного управления
- Все функции доступны через сенсорное управление

## 4. УСТРАНЕНИЕ НЕПОЛАДОК

### 4.1. Проблемы с авторизацией

**Проблема:** Не удаётся войти в систему

**Решение:**
- Проверить правильность ввода email и пароля
- Убедиться, что подключение к интернету активно
- Попробовать восстановить пароль через функцию «Забыли пароль?»
- Очистить кэш браузера и попробовать снова

**Проблема:** Не приходит письмо для восстановления пароля

**Решение:**
- Проверить папку «Спам» в почтовом ящике
- Убедиться, что email введён правильно
- Подождать несколько минут (письмо может прийти с задержкой)
- Попробовать отправить запрос на восстановление пароля снова

### 4.2. Проблемы с воспроизведением музыки

**Проблема:** Трек не воспроизводится

**Решение:**
- Проверить подключение к интернету
- Обновить страницу в браузере
- Проверить, что аудиофайл доступен и не повреждён
- Попробовать воспроизвести другой трек
- Очистить кэш браузера

**Проблема:** Нет звука при воспроизведении

**Решение:**
- Проверить уровень громкости в плеере
- Проверить уровень громкости на устройстве
- Убедиться, что звук не отключён в браузере
- Проверить настройки звука в операционной системе

### 4.3. Проблемы с загрузкой контента

**Проблема:** Не удаётся загрузить трек

**Решение:**
- Убедиться, что пользователь имеет роль «артист»
- Проверить формат аудиофайла (должен быть MP3, WAV, FLAC, OGG или M4A)
- Проверить размер файла (не должен превышать установленные ограничения)
- Проверить подключение к интернету
- Попробовать загрузить файл снова

**Проблема:** Загруженный трек не отображается в списке

**Решение:**
- Обновить страницу в браузере
- Проверить, что трек был успешно загружен (должно быть уведомление об успехе)
- Убедиться, что трек публичный (если нужно, чтобы он отображался в общем каталоге)
- Проверить фильтры и сортировку в разделе «Библиотека»

### 4.4. Проблемы с производительностью

**Проблема:** Приложение работает медленно

**Решение:**
- Проверить скорость подключения к интернету
- Закрыть ненужные вкладки браузера
- Очистить кэш браузера
- Обновить браузер до последней версии
- Попробовать использовать другой браузер

**Проблема:** Страница не загружается

**Решение:**
- Проверить подключение к интернету
- Обновить страницу в браузере (F5 или Ctrl+R)
- Очистить кэш браузера
- Попробовать открыть приложение в режиме инкогнито
- Проверить, что URL-адрес введён правильно

### 4.5. Проблемы с отображением

**Проблема:** Интерфейс отображается некорректно

**Решение:**
- Обновить страницу в браузере
- Очистить кэш браузера
- Проверить, что браузер поддерживается (Google Chrome 90+, Mozilla Firefox 88+, Microsoft Edge 90+, Safari 14+)
- Попробовать изменить масштаб страницы (Ctrl + колесо мыши)
- Попробовать использовать другой браузер

## 5. БЕЗОПАСНОСТЬ И КОНФИДЕНЦИАЛЬНОСТЬ

### 5.1. Защита данных

Приложение использует современные технологии для защиты данных пользователей:

- **Аутентификация** — пароли хэшируются с использованием bcrypt через Supabase Authentication
- **Авторизация** — ролевая модель доступа (RBAC) обеспечивает контроль доступа к функциям приложения
- **Защита данных** — Row Level Security (RLS) политики обеспечивают изоляцию данных пользователей
- **Аудит** — все изменения в критичных таблицах фиксируются в аудит-логе

### 5.2. Конфиденциальность

- Пользовательские данные хранятся в защищённой базе данных Supabase
- Персональные данные не передаются третьим лицам
- Пользователи могут управлять своими данными через раздел «Профиль» и «Настройки»
- Пользователи могут удалить свой аккаунт в любое время

### 5.3. Рекомендации по безопасности

- Используйте надёжный пароль (минимум 8 символов, включая буквы, цифры и символы)
- Не передавайте свои учётные данные третьим лицам
- Регулярно обновляйте пароль
- Выходите из аккаунта при использовании общественных компьютеров
- Сообщайте о подозрительной активности администраторам системы
